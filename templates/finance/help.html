Методичка финансиста: БДР, БДДС, баланс, план-факт

Роль: автоматизация финансовой аналитики, контроль рентабельности и ликвидности.

Задачи финансиста

  • Рассчитывать финансовый результат по направлениям: сегментация продаж и товарных групп
  • Контролировать структуру затрат: прямые (COGS), косвенные, операционные
  • Управлять прогнозом денежных потоков, избегать кассовых разрывов
  • Поддерживать управленческий баланс, сравнивать активы и пассивы
  • Проводить план-фактный анализ, фиксировать причины отклонений

Источники данных

  • SalesOrder/SalesOrderItem/Payment: выручка, сегменты, реальные оплаты
  • PurchaseOrder/PurchaseOrderItem/InventoryBatch: себестоимость, партии, транспорт
  • CashCalendarItem: планирование поступлений/выплат по дням
  • IndirectExpense: аренда, ФОТ, комиссии, налоги, прочие-OPEX
  • Customer/Supplier/Dictionaries: классификация контрагентов и условий оплаты

Ключевая функциональность

  • БДР — сводный орган по выручке/COGS/GP/марже по сегментам
  • БДДС — платежный календарь + 30-дневный прогноз c подсветкой негативных дней
  • Управленческий баланс — активы/пассивы/капитал на любую дату
  • План-факт — сравнение плановых/фактических данных, гибкий бюджет, причины отклонений
  • Executive Dashboard — ключевые KPI (чистая прибыль, cash, ROE, точка безубыточности)

Процессы

  • Ежедневное пополнение данных продаж/закупок и их синхронизация с финансовыми показателями
  • Ежемесячное формирование БДР/БДДС/баланса, ревизия отклонений
  • Факт-план: на совещаниях фиксировать причины и корректировать планы
  • Политика: первые уведомления при отклонении < 20%, аварийные - при кассовых разрывах
