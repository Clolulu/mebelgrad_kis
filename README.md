# Mebelgrad KIS

Flask-приложение CRM/ERP для мебельного бизнеса. Проект поддерживает управление данными, продажи, склад, закупки и финансовое планирование.

## Обзор

Проект включает:
- модуль `data_mdm` для управления товарами, поставщиками, клиентами и сотрудниками;
- финансовый модуль с бюджетированием, косвенными расходами, денежным календарём и балансом;
- демонстрационные сценарии продаж и склада;
- сидирование базы данных реальными демонстрационными данными.

## Технологии

- Python 3.11+ / 3.12+
- Flask
- Flask-Login
- Flask-JWT-Extended
- SQLAlchemy
- SQLite (по умолчанию)

## Установка

1. Клонировать репозиторий:
   ```powershell
   git clone https://github.com/Clolulu/mebelgrad_kis.git
   cd mebelgrad_kis
   ```
2. Создать и активировать виртуальное окружение:
   ```powershell
   python -m venv .venv
   .\.venv\Scripts\activate
   ```
3. Установить зависимости:
   ```powershell
   pip install -r requirements.txt
   ```

## Конфигурация

Для разработки можно создать файл `.env` в корне проекта со следующими переменными:

```env
SECRET_KEY=your_secret_key
JWT_SECRET_KEY=your_jwt_secret_key
DATABASE_URL=sqlite:///instance/mebelgrad_kis.db
FLASK_ENV=development
```

Если `.env` отсутствует, приложение будет работать со стандартными настройками из `config.py`.

## Запуск

```powershell
python run.py
```

Откройте в браузере:

```text
http://127.0.0.1:5000
```

## База данных и данные

- База данных хранится в `instance/mebelgrad_kis.db`.
- Файл базы данных и папка `instance/` игнорируются в `.gitignore`.
- При первом запуске приложение автоматически создаёт схему и наполняет демонстрационными данными.
- Сиды включают реалистичные данные по заказам, закупкам, бюджету, косвенным расходам, денежным календарём и балансу.
- Данные охватывают период до 31 мая 2026 года.

### Обновление/сброс данных

Если нужно пересоздать базу данных:

```powershell
Remove-Item instance\mebelgrad_kis.db
python run.py
```

После этого база будет восстановлена с теми же сидированными данными.

## Полезные файлы

- `app/__init__.py` — инициализация приложения, создание схемы и сидирование данных;
- `app/models.py` — определения моделей данных;
- `run.py` — точка входа приложения;
- `templates/` — HTML-шаблоны интерфейса.

## Примечания

- В репозитории игнорируются файлы `*.db` и директория `instance/`.
- Чтобы другие разработчики получили одинаковые демонстрационные данные, достаточно запустить приложение заново, так как сидирование автоматически создаёт одинаковый набор данных.
